# Learn 탭 — 시각적 알고리즘 학습 공간

Learn 탭은 LeetCode 정복에 필요한 핵심 알고리즘 32개를 단계별 애니메이션으로 배우는 공간입니다. 모든 시각화는 브라우저에서만 동작하며 네트워크·LLM이 필요 없습니다.

## 사용법

- **재생/일시정지**: 자동으로 한 단계씩 진행합니다. 속도는 0.5×~4×.
- **◀ / ▶**: 한 단계씩 앞뒤로 이동합니다. 헷갈리는 순간을 되감아 보세요.
- **셔플(⤨)**: 새로운 무작위 입력으로 처음부터 다시 실행합니다.
- **슬라이더**: 원하는 단계로 바로 점프합니다.
- 오른쪽 **코드 따라가기** 패널에서 지금 실행 중인 줄이 강조되고, 시각화 아래에 현재 변수 값과 한국어 해설이 표시됩니다.

## 효과적으로 공부하는 법

1. **예측하고 재생하기**: 다음 단계에 무슨 일이 일어날지 먼저 말해 본 뒤 ▶를 누르세요. 예측이 틀린 지점이 바로 이해가 빈 곳입니다.
2. **셔플로 일반화 확인**: 한 입력에서 이해했다고 끝내지 말고, 셔플로 다른 입력을 만들어 같은 논리가 통하는지 확인하세요.
3. **핵심 포인트로 마무리**: 각 페이지 하단의 핵심 포인트에는 면접에서 묻는 내용(안정성, 복잡도 퇴화 조건, 불변식)이 정리되어 있습니다.
4. **문제로 연결**: 학습 직후 위 표의 대표 LeetCode 문제를 대시보드의 `Add LeetCode Problem`으로 추가해 직접 풀고, AI 코치에게 힌트·리뷰를 받아 보세요.

## 알고리즘 추가하기 (기여 가이드)

새 알고리즘은 다음 절차를 따릅니다. 자세한 단계는 `.claude/skills/add-learn-algorithm/SKILL.md`를 참고하세요.

1. `apps/web/src/learn/algorithms/`의 알맞은 파일(또는 새 파일)에 `AlgorithmDef`를 작성합니다.
   - 배열 시각화는 `ArrayTracer`(막대 `bars` / 상자 `boxes`), 격자 시각화는 `GridFrame`(셀 라벨·범례 교체 지원)을 사용합니다.
   - 모든 `message`와 `insight`는 한국어로, 사실에 근거해 작성합니다.
2. `apps/web/src/learn/index.ts`의 레지스트리에 등록합니다.
3. **필수**: `apps/web/scripts/verify-learn.ts`에 독립적인 레퍼런스 구현 기반 검증 루틴을 추가합니다. 검증이 없는 알고리즘은 감사 스크립트가 실패시킵니다.
4. `pnpm test`로 무작위 입력 40회 × 전 알고리즘 감사를 통과시킵니다.

## 정확성 감사(audit) 체계

`apps/web/scripts/verify-learn.ts`는 `pnpm test`마다 실행되며 다음을 보장합니다.

- 시각화 시뮬레이션의 최종 결과가 **독립적인 레퍼런스 구현**(브루트포스 완전 탐색 포함)과 일치
- 예: 배낭 문제는 2ⁿ 부분집합 완전 탐색과, 카데인은 O(n²) 브루트포스와, 다익스트라는 별도 구현과 대조
- 모든 프레임의 코드 라인 참조가 표시 코드 범위 안에 있고, 포인터가 배열 범위를 벗어나지 않음
- 알고리즘마다 무작위 입력으로 40회 반복 실행

검증 루틴이 등록되지 않은 알고리즘이 레지스트리에 있으면 감사가 실패하므로, 검증 없이 알고리즘을 추가할 수 없습니다.
